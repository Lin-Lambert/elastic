# Markdown 写作与排版技巧

## 目的

提供一套结构感强、视觉舒适的 Markdown 写作规范，确保技术文档具备专业性和可读性。

## 核心原则

- **通俗易懂且整洁专业**：避免将 Markdown 当作简单的记事本，追求结构感、呼吸感和视觉焦点。

## 排版规范

### 1. 中英文混排

在中文与英文字母、数字之间强制插入一个半角空格。

- 正确：`使用 Elasticsearch 进行全文搜索`
- 错误：`使用Elasticsearch进行全文搜索`

### 2. 专有名词

变量名、文件名、命令等必须使用内联代码（`` `Inline Code` ``）包裹。

- 正确：打开 `elasticsearch.yml` 配置文件
- 错误：打开 elasticsearch.yml 配置文件

### 3. 视觉题眼

使用引用块（`>`）标记核心概念、注意事项或比喻总结，形成视觉焦点。

```markdown
> **核心概念**：倒排索引是 Elasticsearch 的基石。
```

### 4. 层级控制

- 仅使用一个 `#`（文档标题）。
- 核心结构使用 `##`（大章节）和 `###`（小节）。
- 对于重要的细分逻辑化内容（如特定的技术点、分类项），使用 `####` 标题并附带数字编号（如 `2.1`），并同步至文档顶部的悬浮大纲。
- 仅对于单行、偏辅助解释的内容，退而使用列表并在开头加粗提词。

### 5. 代码块

代码块必须带上语言标识，如 `json`、`yaml`、`log`、`bash`、`java` 等。

````markdown
```json
{"name": "elasticsearch"}
```
````

## 图文结合

### 图前铺垫

在图表前用 1-2 句话交代背景，说明图表的目的或要解决的问题。

### 图后总结

在图表后紧接有序列表拆解运行过程或关键要点。

## 目录规范

- **触发条件**：文档超过 3 个二级标题（`##`）时。
- **位置**：H1 标题下方，使用 `## 目录`。
- **层级**：收录 `##` 和 `###`。
- **格式**：使用 Markdown 锚点链接。
